# Lessons Learned & Technical Guidelines

This document tracks historical development mistakes, regressions, and design discrepancies to prevent identical issues as the application scales. Engineers should review these before modifying core structural logic.

## 1. Type and Property Mismatches
**The Mistake**: Supplying an object to a component property that explicitly required a list (Array). For example, passing `BoxShadow` directly into an overarching layout container modifier rather than wrapping it like `[BoxShadow(...)]` caused application-breaking `AttributeError`/Type errors.
**The Remedy**: Always scrutinize the API parameters when upgrading libraries or adopting new Shadcn/Radix abstractions.

## 2. Incorrect Import Scopes
**The Mistake**: Importing properties from nested modules incorrectly instead of the expected export location (e.g., `Alignment` imported off an inner module rather than root).
**The Remedy**: In modern TypeScript/Vite React applications, stick exclusively to the export patterns generated by the toolchain. Let the bundler dictate root-level exports. (i.e. favor `@/components/ui/button` rather than pulling directly from Radix internally unless necessary).

## 3. Dynamic Icons and Naming Conventions
**The Mistake**: Using deprecated or incorrectly suffixed icon handles (e.g., using `ANALYTICS` instead of `ANALYTICS_ROUNDED`) resulting in rendering failures in the UI. 
**The Remedy**: When utilizing the `lucide-react` icon library, ensure you trace the exact icon key directly against their current version registry. React will throw if missing icons are instantiated as components.

## 4. UI Polish & Alignment Synchronizations
**The Mistake**: Allowing tabular data/headers to fall out of flex alignment during complex screen transitions or on edge-case viewport sizes. Unchecked pixel alignments damaged the premium aesthetic.
**The Remedy**: Employ flexible, relative Tailwind classes (`flex-1`, `w-full`, standard explicit `gap-*`) instead of hardcoding absolute pixel widths (`width: 324px`) when synchronizing headers with data bodies. Ensure robust contrast across light/dark themes.

## 5. Lifecycle Initialization
**The Mistake**: Forcing settings (such as full-screen overrides or default routing modes) too late in the lifecycle, causing jittering layout shifts.
**The Remedy**: Establish dominant layouts or viewport behaviors (like configuring full-screen Web App manifests) exactly at the main component mount or via `index.html`/`vite-env.d.ts` before React reconciles the tree.
